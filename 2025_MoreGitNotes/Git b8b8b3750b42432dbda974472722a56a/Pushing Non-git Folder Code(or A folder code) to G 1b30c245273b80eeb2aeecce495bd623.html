Pushing Non-git Folder Code(or A folder code) to Git Repo(or B Git Repo)(oops i did it again)

Logic

[Step1]Now we want to do one thing of the following first

  • Initialize this folder and set to an existing Git Repo
  • Reset the connected Git repo to another Git repo

Because we have code in this current folder that may be different from this new remote Git Repo so we need to solve conflict before pushing

[Step2]we first do the following

git pull origin main --allow-unrelated-histories

What Happens Internally?

  1. Git downloads the remote repository's latest changes.
  1. It creates a merge commit combining your local files and remote files.
  1. If there are conflicts, Git will notify you to resolve them.

[Step3]After solving all conflicts, push normally

git push origin main

Step by Step in Details

Step1

Initialize git and set it to connect with an existing Git repo

git init
git remote add origin git@github.com:username/repository.git
## "git@github.com:username/repository.git" this part is from github ssh url

Set current git folder to another existing remote Git repo

git remote set-url origin git@github.com:pristinawang/table_qa.git
## This "git@github.com:pristinawang/table_qa.git" part is the ssh url on Github

Step2

I have set the default strategy as “merge” for transparency log so now when conflicts happen during pull, “merge” will be chosen as the strategy to use

Run this to first pull all changes from remote 

git pull origin main --allow-unrelated-histories
